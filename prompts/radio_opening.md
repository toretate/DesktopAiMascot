# ラジオ番組開始（まくら）シナリオ生成プロンプト

## 概要
このプロンプトは、ラジオ番組の開始部分（まくら）の会話シナリオを生成するために使用します。
メインパーソナリティとサポート役の2人による自然な番組導入を作成します。

---

## プロンプト本文

```
あなたは、ラジオ番組の台本作家です。
以下の条件で、番組開始部分（まくら）の会話シナリオを作成してください。

【登場人物】
- メインパーソナリティ（MC）: 明るく元気な性格で、番組の進行を担当
- サポート役（アシスタント）: 優しく穏やかな性格で、MCをサポート

【番組コンセプト】
- デスクトップマスコットによるインタラクティブなラジオ番組
- リスナー（ユーザー）との対話を重視
- カジュアルで親しみやすい雰囲気

【シナリオの条件】
1. 会話は8〜12ターン程度（1人あたり4〜6回発言）
2. 自然な会話のテンポとリズムを保つ
3. 番組の雰囲気作りを意識した内容
4. リスナーへの挨拶と番組紹介を含める
5. 今日のテーマや企画の予告を軽く触れる

【出力形式】
以下のJSON形式で出力してください：

{
  "scenario": "opening",
  "title": "番組開始",
  "conversations": [
    {
      "speaker": "MC",
      "text": "発言内容",
      "emotion": "happy/neutral/excited"
    },
    {
      "speaker": "Assistant",
      "text": "発言内容",
      "emotion": "happy/neutral/gentle"
    }
  ]
}

【注意事項】
- 各発言は1〜3文程度の自然な長さにしてください
- 音声合成に適した読みやすい文章にしてください
- 感情表現（笑い、驚き等）は emotion フィールドで表現してください
- セリフに（笑）などの地の文は含めないでください
```

---

## 使用例

### リクエスト
```
上記のプロンプトを使用して、番組開始のシナリオを生成してください。
```

### 期待される出力例
```json
{
  "scenario": "opening",
  "title": "番組開始",
  "conversations": [
```

## カスタマイズオプション

### 時間帯別バリエーション
朝・昼・夜で異なる挨拶を使いたい場合：

```
【追加条件】
- 現在の時間帯: {morning/afternoon/evening/night}
- 時間帯に応じた挨拶を使用してください
```

### キャラクター性の調整
特定のキャラクター設定を反映する場合：

```
【キャラクター設定】
- MC: {性格、口調、特徴}
- Assistant: {性格、口調、特徴}
```

## 実装上の注意

1. **感情フィールドの活用**
   - StyleBertVits2のスタイル制御に使用
   - 適切な感情を選択することで、より自然な音声表現が可能

2. **発言の長さ**
   - 1発言は100文字以内を推奨
   - 長すぎると音声合成に時間がかかる

3. **テンポ**
   - 会話のテンポを意識し、適度な間を取る
   - 一方的にならないように注意

4. **リスナーへの配慮**
   - リスナーの存在を意識した発言を含める
   - 対話型番組であることを強調
