# マスコット表情スプライト自動位置合わせ仕様書

## 1. 概要

### 1.1 背景

現状、AI生成またはインポートされた表情スプライトは、マスコット本体（衣装画像）の上に重ねて表示される際、ユーザーが手動で `offsetX`、`offsetY`、`scale` をスライダー操作で調整する必要がある。この作業は28種類の感情スロットそれぞれに対して個別に必要となり、特にキャラクターの顔の位置やサイズが衣装画像ごとに異なるため、非常に手間がかかる。

### 1.2 目的

本機能は、表情スプライト画像とベース衣装画像（全身像）の両方を画像解析し、表情画像を自動的にマスコットの「顔」の位置・サイズに合わせて配置するためのパラメータ（`offsetX`, `offsetY`, `scale`）を自動算出する。

### 1.3 対象スコープ

以下の操作時に自動位置合わせを適用する。

1. **AIスプライトシートのインポート時** — スプライトシートから切り出した各表情画像を一括で自動位置合わせ
2. **手動での表情スロット割り当て時** — 個別のスロットに画像を割り当てた直後に自動位置合わせ
3. **衣装（Outfit）切り替え時** — 新しい衣装に対して既存の表情画像の位置を再計算
4. **表情エディタからのワンクリック自動調整** — 「自動位置合わせ」ボタンで手動で起動

### 2.1 処理パイプライン

```
┌────────────────────────────────────────────────────────────────────┐
│ 自動位置合わせパイプライン                                           │
│                                                                    │
│  [入力]                                                            │
│    ├─ ベース画像（衣装/ポーズ全身像）                                 │
│    └─ 表情画像（顔スプライト）                                       │
│                                                                    │
│  [Step 1] 顔領域検出 (FaceRegionDetector)                          │
│    ├─ ベース画像 → 顔バウンディングボックス取得                       │

│                                                                    │
│  [Step 2] 表情画像の有効領域検出 (ContentBoundsDetector)              │
│    └─ 表情画像 → 非透明ピクセルのバウンディングボックス               │
│                                                                    │
│  [Step 3] パラメータ算出 (AlignmentCalculator)                      │
│    ├─ scale = ベース顔幅 / 表情有効幅                               │

│    └─ 半透明グラデーションマスキング（エッジスムージング）             │
│                                                                    │
│  [出力]                                                            │
│    ├─ offsetX, offsetY, scale                                      │
│    └─ (オプション) マスキング済み画像の Base64                       │
└────────────────────────────────────────────────────────────────────┘
```

## 3. 各モジュール詳細設計

### 3.1 顔領域検出 (FaceRegionDetector)

#### 目的

ベース画像（衣装全身像）のどの位置に「顔」があるかを推定し、バウンディングボックスとして返す。

#### アルゴリズム

ブラウザ環境（Canvas API）で動作する軽量な画像解析を用いる。外部ライブラリや API への依存を避けるため、ヒューリスティックベースのアプローチを採用する。

**Phase 1: 非透明領域の検出**

既存の `getNonTransparentBounds` ロジック（[MascotViewer.vue](../../ui/src/components/MascotViewer.vue) L255-293）を再利用する。

```typescript
interface BoundingBox {
    top: number;    // Y座標（px）
    bottom: number; // Y座標（px）
    left: number;   // X座標（px）
    right: number;  // X座標（px）
}
```

**Phase 2: 顔領域の推定**

キャラクターイラストの一般的な比率に基づき、非透明領域の上部に「顔」が位置すると仮定する。

```typescript
/**
 * 顔領域の推定ヒューリスティック:
 * 
 * - 顔の縦方向位置: 非透明領域の上端から全体高さの 10%〜35% の範囲
 * - 顔の横方向位置: 非透明領域の水平中心を基準に、幅の 25% を左右に取る
 * - 顔の高さ: 全体高さの約 25%
 * - 顔の幅: 全体幅の約 50%
 */
```

この推定は多くのアニメ/イラスト調のキャラクターで有効だが、特殊なポーズ（しゃがみ、横臥等）では精度が低下するため、ユーザーによる手動補正が引き続き利用可能であることを保証する。

**Phase 3 (将来拡張): Gemini Vision による精密検出**

現在のスプライトシート解析で使用している Gemini Vision API を転用し、ベース画像に対して「顔のバウンディングボックス」を返すリクエストを行うオプションを用意する。

```typescript
interface FaceDetectionResult {
    faceBox: BoundingBox;        // 検出された顔の領域
    confidence: number;          // 信頼度 (0.0 - 1.0)
    method: 'heuristic' | 'ai'; // 使用された検出手法
}
```

#### インターフェース

```typescript
export class FaceRegionDetector {
    /**
     * ベース画像から顔領域を検出する
     * @param imageSource 画像パスまたは Base64 Data URL
     * @returns 顔のバウンディングボックスと検出メタデータ
     */
    static async detect(imageSource: string): Promise<FaceDetectionResult>;

    /**
     * Gemini Vision API を使った高精度検出
     * @param imageSource 画像パスまたは Base64 Data URL
     * @param apiKey Gemini API キー
     * @returns 顔のバウンディングボックス
     */
    static async detectWithAI(imageSource: string, apiKey: string): Promise<FaceDetectionResult>;
}
```

### 3.2 表情画像の有効領域検出 (ContentBoundsDetector)

#### 目的

表情スプライト画像の中で実際にコンテンツ（非透明ピクセル）が存在する領域のバウンディングボックスを検出する。これにより、余白部分を除いた「実際の顔画像サイズ」が得られる。

#### アルゴリズム

```typescript
/**
 * Canvas API を用いた非透明ピクセルのバウンディングボックス検出
 *
 * 1. 画像を Canvas に描画
 * 2. getImageData() で全ピクセルデータを取得
 * 3. alpha > threshold (デフォルト: 10) のピクセルを走査
 * 4. minX, minY, maxX, maxY を記録
 * 5. 白色背景の場合は色差判定を併用:
 *    - RGB 各チャンネルが (250, 250, 250) 以上かつ alpha > 250 の場合は「背景」として除外
 */
```

#### 背景色の自動検出

多くのAI生成スプライトシートは白背景（`#ffffff`）で出力される。背景色を自動検出して有効領域の判定に利用する。

```typescript
/**
 * 背景色の自動検出:
 * 
 * 1. 画像四隅のピクセル（各隅 5x5px の領域）の色をサンプリング
 * 2. 四隅の色の平均が十分に近い場合（ΔE < 15）、それを背景色として採用
 * 3. 背景色からの色差が閾値以下のピクセルは「背景」として扱う
 */
```

#### インターフェース

```typescript
export interface ContentBounds {
    box: BoundingBox;
    imageWidth: number;
    imageHeight: number;
    centerX: number;  // 有効領域の中心X
    centerY: number;  // 有効領域の中心Y
    contentWidth: number;   // 有効領域の幅
    contentHeight: number;  // 有効領域の高さ
}

export class ContentBoundsDetector {
    /**
     * 画像の有効領域（非透明・非背景のバウンディングボックス）を検出する
     * @param imageSource 画像パスまたは Base64 Data URL
     * @param backgroundColorThreshold 背景色との色差閾値（デフォルト: 30）
     */
    static async detect(imageSource: string, backgroundColorThreshold?: number): Promise<ContentBounds>;
}
```

### 3.3 パラメータ算出 (AlignmentCalculator)

#### 目的

顔領域検出結果と表情画像の有効領域情報をもとに、表情エディタの `offsetX`、`offsetY`、`scale` パラメータを算出する。

#### 座標系

表情エディタ（[ExpressionEditorModal.vue](../../ui/src/components/settings/ExpressionEditorModal.vue)）のプレビューは以下の座標系で動作する。

- プレビュー領域サイズ: **420 × 560 px**
- 表情画像の基本描画サイズ: **140 × 140 px**
- transform の原点: プレビュー領域の **中央**（`position: absolute` + flexbox centering）
- offset パラメータ: プレビュー領域中央からのピクセルオフセット（-250 〜 250）
- scale パラメータ: 140px 基準の倍率（0.3 〜 2.0）

#### 算出ロジック

```typescript
interface AlignmentParams {
    offsetX: number;   // -250 〜 250 (px)
    offsetY: number;   // -250 〜 250 (px)
    scale: number;     // 0.3 〜 2.0
}

/**
 * 算出手順:
 *
 * 1. ベース画像をプレビュー領域にフィットさせた場合のスケール比率を算出
 *    fitScale = min(420 / baseImageWidth, 560 / baseImageHeight)
 *
 * 2. 顔領域のプレビュー座標系への変換
 *    faceCenterX_preview = (faceBox.left + faceBox.right) / 2 * fitScale
 *    faceCenterY_preview = (faceBox.top + faceBox.bottom) / 2 * fitScale
 *    faceWidth_preview = (faceBox.right - faceBox.left) * fitScale
 *
 * 3. offset の算出
 *    previewCenterX = 420 / 2 = 210
 *    previewCenterY = 560 / 2 = 280
 *    
 *    // ベース画像のフィット時のオフセット補正
 *    baseOffsetX = (420 - baseImageWidth * fitScale) / 2
 *    baseOffsetY = (560 - baseImageHeight * fitScale) / 2
 *    
 *    offsetX = (faceCenterX_preview + baseOffsetX) - previewCenterX
 *    offsetY = (faceCenterY_preview + baseOffsetY) - previewCenterY
 *
 * 4. scale の算出
 *    // 表情画像の有効領域を顔領域にフィットさせるための倍率
 *    contentScale = faceWidth_preview / contentBounds.contentWidth
 *    // 140px 基準の表情画像サイズに正規化
 *    scale = contentScale * (contentBounds.imageWidth / 140)
 *    // 表情スプライトは「顔」のみを含むため、顔領域幅の 80% にフィットさせる
 *    scale *= 0.8
 *
 * 5. クランプ
 *    offsetX = clamp(offsetX, -250, 250)
 *    offsetY = clamp(offsetY, -250, 250)
 *    scale = clamp(scale, 0.3, 2.0)
 */
```

#### インターフェース

```typescript
export class AlignmentCalculator {
    /**
     * 顔領域と表情画像の有効領域からオフセット・スケールを算出
     * 
     * @param faceDetection ベース画像の顔検出結果
     * @param contentBounds 表情画像の有効領域
     * @param baseImageSize ベース画像のネイティブサイズ { width, height }
     * @param previewSize プレビュー領域サイズ（デフォルト: { width: 420, height: 560 }）
     * @param expressionBaseSize 表情画像の基本描画サイズ（デフォルト: 140）
     */
    static calculate(
        faceDetection: FaceDetectionResult,
        contentBounds: ContentBounds,
        baseImageSize: { width: number; height: number },
        previewSize?: { width: number; height: number },
        expressionBaseSize?: number
    ): AlignmentParams;
}
```

### 3.4 背景マスキング (BackgroundMasker)

#### 目的

AI生成されたスプライトは多くの場合、白色やグレーの単色背景を含んでおり、マスコット本体の上に重ねた際に背景部分が不自然に見える。この問題を解決するために、背景を自動除去する。

#### モード

| モード | 説明 | ユースケース |
|---|---|---|
| **ハードマスキング** | 背景色と判定されたピクセルの alpha を完全に 0 にする | 単色背景が明確な場合 |
| **ソフトマスキング** | 背景色からの色差に応じてグラデーション的に alpha を減衰させる | エッジにアンチエイリアスがかかっている場合 |
| **エッジスムージング** | 有効領域の境界から外側 N px にかけて alpha を徐々に下げる | 切り抜き境界をなめらかにしたい場合 |

#### アルゴリズム

```typescript
/**
 * 背景マスキング処理:
 *
 * 1. 背景色の特定
 *    - ContentBoundsDetector の背景色検出結果を利用
 *    - ユーザー指定がある場合はそれを優先
 *
 * 2. ハードマスキング
 *    for each pixel:
 *      colorDistance = sqrt((r - bgR)^2 + (g - bgG)^2 + (b - bgB)^2)
 *      if colorDistance < hardThreshold:
 *        alpha = 0
 *
 * 3. ソフトマスキング (ハードマスキングの外周)
 *    for each pixel:
 *      if hardThreshold <= colorDistance < softThreshold:
 *        alpha = alpha * ((colorDistance - hardThreshold) / (softThreshold - hardThreshold))
 *
 * 4. エッジスムージング
 *    有効領域の外縁から内側に向かって gradientWidth (px) 幅で
 *    alpha をリニアに 0 → 元の alpha へグラデーションする
 */
```

#### インターフェース

```typescript
export interface MaskingOptions {
    /** マスキングモード */
    mode: 'hard' | 'soft' | 'edge-smooth';
    /** ハードマスキングの色差閾値（デフォルト: 30） */
    hardThreshold?: number;
    /** ソフトマスキングの色差閾値（デフォルト: 80） */
    softThreshold?: number;
    /** エッジスムージングの幅 (px)（デフォルト: 3） */
    edgeSmoothWidth?: number;
    /** 手動指定の背景色 [R, G, B]（省略時は自動検出） */
    backgroundColor?: [number, number, number];
}

export class BackgroundMasker {
    /**
     * 画像の背景をマスキングして新しい Base64 画像を返す
     * @param imageSource 画像パスまたは Base64 Data URL
     * @param options マスキングオプション
     * @returns マスキング済み画像の Base64 Data URL
     */
    static async mask(imageSource: string, options?: MaskingOptions): Promise<string>;
}
```

### 3.5 メインオーケストレーター (ExpressionAutoAligner)

全モジュールを統合し、一連の自動位置合わせ処理をワンコールで実行するファサード。

#### インターフェース

```typescript
export interface AutoAlignOptions {
    /** 背景マスキングを適用するか（デフォルト: true） */
    applyMasking?: boolean;
    /** マスキングオプション */
    maskingOptions?: MaskingOptions;
    /** AI検出を使用するか（デフォルト: false。true の場合 apiKey が必要） */
    useAIDetection?: boolean;
    /** Gemini API キー（AI検出時に必要） */
    apiKey?: string;
}

export interface AutoAlignResult {
    /** 算出された位置パラメータ */
    params: AlignmentParams;
    /** マスキング済み画像の Base64（マスキング適用時のみ） */
    maskedImage?: string;
    /** 顔検出の信頼度 */
    confidence: number;
    /** 使用された検出手法 */
    method: 'heuristic' | 'ai';
}

export class ExpressionAutoAligner {
    /**
     * 単一の表情画像を自動位置合わせする
     */
    static async alignSingle(
        baseImage: string,
        expressionImage: string,
        options?: AutoAlignOptions
    ): Promise<AutoAlignResult>;

    /**
     * 複数の表情画像を一括で自動位置合わせする
     * ベース画像の顔検出は1回だけ実行し、結果を共有する
     */
    static async alignBatch(
        baseImage: string,
        expressionImages: { id: string; image: string }[],
        options?: AutoAlignOptions
    ): Promise<Map<string, AutoAlignResult>>;
}
```

```
  切り出し → スロット割り当て → 感情割り当て画面
```

## 5. 設定と調整パラメータ

### 5.1 ヒューリスティック調整定数

以下のパラメータは、キャラクターのスタイルやポーズに応じて調整が必要になる可能性がある。将来的にユーザー設定として公開することを検討する。

| 定数 | デフォルト値 | 説明 |
|---|---|---|
| `FACE_TOP_RATIO` | 0.10 | 非透明領域の上端からの顔開始位置（比率） |
| `FACE_BOTTOM_RATIO` | 0.35 | 非透明領域の上端からの顔終了位置（比率） |
| `FACE_WIDTH_RATIO` | 0.50 | 非透明領域の幅に対する顔幅の比率 |
| `FACE_FIT_RATIO` | 0.80 | 表情画像を顔領域の何割にフィットさせるか |
| `BG_COLOR_THRESHOLD` | 30 | 背景色判定の色差閾値 |
| `ALPHA_THRESHOLD` | 10 | 透明判定の alpha 閾値 |

### 5.2 デスクトップウィンドウへのスケール補正

表情エディタで算出されたパラメータは、デスクトップウィンドウ（512px）で表示する際に補正係数 `512 / 420 ≈ 1.219` が自動適用される（既存の[MascotViewer.vue](../../ui/src/components/MascotViewer.vue) L233 のロジック）。自動位置合わせモジュールはエディタ座標系（420px ベース）でパラメータを算出するため、この補正は変更不要。

## 6. エラーハンドリングとフォールバック

| シナリオ | 対応 |
|---|---|
| ベース画像が空/絵文字のみ | 自動位置合わせをスキップし、デフォルト値 (0, 0, 1.0) を維持 |
| 表情画像が空/未登録 | 処理対象から除外 |
| 顔領域検出に失敗（全面透明等） | ベース画像の上部 1/3 中央をフォールバック顔領域とする |
| AI検出の API エラー | ヒューリスティック検出にフォールバック |
| 算出値が範囲外 | クランプ (offsetX/Y: -250〜250、scale: 0.3〜2.0) |
| 背景色検出不能 | マスキング処理をスキップし、元画像をそのまま使用 |
