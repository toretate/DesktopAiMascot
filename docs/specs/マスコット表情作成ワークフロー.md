# マスコットの表情作成ワークフロー

この資料は、マスコットの表情作成およびアニメーション構築、アセット最適化のワークフローについて記述します。
自動処理と並行して、ユーザーが作成状況を把握し、プレビューを見ながら手動で位置・拡大縮小を調整できる「ユーザー介入ステップ」を含んでいます。

---

## 1. 表情スプライト・位置合わせワークフロー

表情スプライトは、ベース画像（衣装・ポーズ）との輪郭のズレによるノイズを防ぐため、**「位置合わせを完了させてからパーツ（目・眉・口・鼻）のみを抽出する」** 順序で処理を行います。

### 1.1 のっぺらぼう（ベース画像）の準備
1. **ベース画像の規格化**
    * ベース画像をアスペクト比一定のまま、`w:h = 1152:1920` （`(768x1280) x 1.5`）になるよう拡大または縮小します。
2. **背景透過処理**
    * 背景を削除し、必要に応じて白（`#FFF`）で塗りつぶす等の処理を行います。
3. **顔の検出と切り抜き**
    * ベース画像から顔の境界を検出し、顔領域を矩形または楕円で切り抜きます。
4. **「のっぺらぼう」画像の生成**
    * 切り抜いた顔画像から目、眉、口、鼻を消去した「のっぺらぼう」画像を生成します。これが表情パーツを重ねる土台になります。
5. **【ユーザー確認・調整】のっぺらぼう画像の確認**
    * **表示**: 生成されたのっぺらぼう画像のプレビューを表示します。
    * **介入**: 目や口の消し残しがある場合、簡易ブラシツールでの修正や、再生成の要求を行えます。

### 1.2 表情画像の生成と位置合わせ（アライメント）
1. **AI表情スプライトの生成**
    * AI等を用いて表情差分画像を生成します。
    * （例：各表情を1画像内で4表情ずつ並べた `1536x2560` のスプライトシートとして生成し、これを縦横方向に1/2して各表情スプライト `768x1280` を作成）
2. **表情画像の位置合わせ（レジストレーション）**
    * 生成された表情画像（顔全体）を、ベースの「のっぺらぼう」画像に対して位置合わせします。
    * OpenCV等を用いて、回転（`rotation`）、縮尺（`scale`）、平行移動（`offsetX`/`offsetY`）を算出・適用します。
3. **【ユーザー確認・調整】配置位置・拡大縮小の手動調整**
    * **表示**: のっぺらぼう画像の上に、自動位置合わせを適用した表情画像を半透明で重ねてプレビュー表示します。
    * **介入**: 自動合わせがズレている場合、プレビュー上でのドラッグ操作や、設定画面のスライダー（`offsetX`, `offsetY`, `scale`, `rotation`）を操作して、表情パーツの配置を微調整します。

### 1.3 表情パーツの抽出
1. **変形適用後の差分（XOR）抽出**
    * 位置合わせした後の表情画像と、ベースののっぺらぼう画像の差分（XOR等）を処理します。
    * あらかじめ完全に位置を重ねてから差分を取ることで、顔の輪郭や髪のズレによる境界ノイズを排除し、純粋な「目・眉・鼻・口」の表情パーツのみを綺麗に抽出します。
2. **マスク処理**
    * 目・眉・鼻・口の領域検出を行い、必要な表情パーツ部分のみを残すためのアルファマスクを適用して切り抜きます。

## 2. 瞬き（目）と口パク（口）の作成ワークフロー

感情の豊かさとアニメーションの組み合わせ爆発を防ぐため、**「目・眉（Eyes）」パーツと「口・鼻（Mouth）」パーツを完全に分離して管理**します。

### 2.1 瞬き（目・眉）アニメーション
1. 各感情（joy, sadness 等）から抽出した「目・眉」パーツをベースにします。
2. 瞬きアニメーション用に、以下の3状態（またはそれ以上）の差分フレームを生成します。
    * **開（Open）**
    * **半閉（Half-Closed）**
    * **閉（Closed）**

### 2.2 口パク（口・鼻）アニメーション
1. 通常表情または各感情から抽出した「口・鼻」パーツをベースにします。
2. 音声同期（リップシンク）用に、日本語の母音をベースとした以下の差分フレームを生成します。
    * **閉（Closed）**
    * **あ（A）**
    * **い（I）**
    * **う（U）**
    * **え（E）**
    * **お（O）**

### 2.3 【ユーザー確認・調整】アニメーション動作プレビュー
* **表示**: 分離したパーツをのっぺらぼうの上に配置し、瞬き・口パクが動いているプレビューアニメーションをその場でループ再生して表示します。
* **介入**: 瞬きの速度調整や、特定のフレーム（半閉など）の位置に微小なズレがある場合、プレビューを確認しながらフレームごとの配置オフセットを微調整できます。

### 2.4 描画時のレイヤー合成
実行時は、ベースとなる「のっぺらぼう」画像の上に、以下のレイヤーを順に重ね合わせて動的に合成描画（CSSの重ね合わせまたはPixiJSによる描画）します。
1. **ベースレイヤー**: のっぺらぼう画像
2. **目レイヤー**: 現在の感情の「目・眉」パーツ（瞬きアニメーションの現在のフレーム）
3. **口レイヤー**: 現在の感情または発話状態に合わせた「口・鼻」パーツ（口パクの現在のフレーム）

## 3. アセット最適化（テクスチャアトラス化）ワークフロー

数百枚に及ぶ差分パーツ画像（28感情 × 瞬き・口パクフレーム）を個別に保存すると、ファイル読み込みのI/Oオーバーヘッドやメモリ消費が大きくなるため、**テクスチャアトラス形式（SpineのAtlas等と同様の構造）**にパッキングして管理します。

### 3.1 アトラス画像（`atlas.png`）の生成
* 切り出された「のっぺらぼう」「目（各感情・各瞬きフレーム）」「口（各口パクフレーム）」などの個別スプライトを、1枚または数枚の大きなテクスチャ（例: `2048x2048`）に隙間なく敷き詰めてパッキングし、1枚の画像として書き出します。

### 3.2 座標定義ファイル（`atlas.json`）の生成
* アトラス画像内のどの位置にどのパーツがあるかを定義するメタデータJSONを生成します。

```json
{
    "meta": {
        "image": "atlas.png",
        "size": { "w": 2048, "h": 2048 }
    },
    "frames": {
        "neutral_eyes_open": {
            "frame": { "x": 0, "y": 0, "w": 180, "h": 120 },
            "pivot": { "x": 90, "y": 60 },
            "offset": { "x": 10, "y": -50 }
        },
        "neutral_eyes_closed": {
            "frame": { "x": 180, "y": 0, "w": 180, "h": 120 },
            "pivot": { "x": 90, "y": 60 },
            "offset": { "x": 10, "y": -50 }
        },
        "mouth_a": {
            "frame": { "x": 0, "y": 120, "w": 100, "h": 80 },
            "pivot": { "x": 50, "y": 40 },
            "offset": { "x": 0, "y": 80 }
        }
    }
}
```

* **プロパティ説明**:
    * `frame`: アトラス画像上での切り出し矩形（`x, y, w, h`）
    * `pivot`: パーツ自体の中心位置座標
    * `offset`: のっぺらぼうの基準顔中心位置に対する、パーツの重畳配置オフセット位置（`offsetX, offsetY`）

### 3.3 【最終確認】パッキングアセットのプレビュー
* アトラスおよびJSONの保存前に、ユーザーに対してパッキング結果の概要を表示し、最終的な確認・保存の完了を行えるようにします。
