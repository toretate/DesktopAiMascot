# マスコット表情仕様書

## 概要

本ドキュメントでは、デスクトップAIマスコットにおける表情システムの仕様を記述する。  
表情システムは以下の4つの主要コンポーネントで構成される。

1. **感情定義** — サポートする感情の一覧とそのメタデータ
2. **スプライト自動生成** — AIを用いた表情スプライトシートの自動生成
3. **表情割り当て** — スプライトシートから各感情スロットへの自動・手動割り当て
4. **表情位置合わせ** — 表情画像のマスコット本体への重ね合わせ位置調整

## 2. 画像アセット構造 (MascotImageSet)

### 2.1 画像セットの種別

表情画像は `MascotImageSet` クラス（[MascotImageSet.ts](../../ui/src/mascots/MascotImageSet.ts)）で管理され、3種類の画像カテゴリに分類される。

| プロパティ | 役割 |
|---|---|
| `image` | 画像セットの代表画像（カバー） |
| `angleImages` | 方向別画像 (`front`, `left`, `right`, `back`, `top`, `bottom` 等) |
| `emotionFaceImages` | 感情別の顔アップ画像（28感情） |
| `emotionFullbodyImages` | 感情別の全身画像（28感情） |

### 2.2 ファイル名命名規則 (MascotImageSetBuilder)

`MascotImageSetBuilder`（[MascotImageSetBuilder.ts](../../ui/src/mascots/MascotImageSetBuilder.ts)）はファイル名のサフィックスにより画像を自動分類する。

#### 方向画像 (Angle)

ファイル名の末尾トークン（`_` 区切り）が以下のいずれかに一致する場合、`angleImages` として分類される。

```
left, right, front, back, top, bottom, above, below, behind
```

**例：** `mascot_front.png` → `angleImages["front"]`

#### 感情・顔アップ画像 (EmotionFace)

末尾から2番目のトークンが `face` **または** 末尾トークンが `face` の場合、感情名をキーとして `emotionFaceImages` に分類される（トークン数は3以上必要）。

**例：** `chara_face_joy.png` → `emotionFaceImages["joy"]`  
**例：** `chara_joy_face.png` → `emotionFaceImages["joy"]`

#### 感情・全身画像 (EmotionFullbody)

末尾から2番目のトークンが `fullbody` または `body` **または** 末尾トークンが同じ値の場合、`emotionFullbodyImages` に分類される。

**例：** `chara_fullbody_joy.png` → `emotionFullbodyImages["joy"]`  
**例：** `chara_joy_fullbody.png` → `emotionFullbodyImages["joy"]`

#### カバー画像の除外

ファイル名が `cover`（大文字小文字問わず）の場合は分類対象から除外される。

#### フォールバック

上記のいずれにも該当しない場合は、代表画像 (`image`) として扱われる。

### 2.3 実際のアセット格納先

```
mascots/<mascot_id>/
    expressions/
        expr_joy.png
        expr_anger.png
        expr_normal.png
        ...
    outfits/
        outfit_<id>.png
    avatar/
    poses/
```

## 3. スプライト自動生成

### 3.1 生成フロー

表情スプライトシートの自動生成は `AiExpressionService`（[expression-service.ts](../../ui/src/skills/expression-service/expression-service.ts)）が担当する。

```
[ユーザーが生成する表情を選択（最大16個）]
        ↓
[ベースキャラクター画像の取得]
  優先順位:
    1. 現在アクティブな衣装（outfit）
    2. 全身像の最初のアセット
    3. 正面（front）アバター
    4. ベースアバター
        ↓
[感情リストとプロンプトを生成]
  - 選択された感情の英語ラベルをリスト化
  - common プロンプト + ユーザープロンプト + ラベル指示プロンプトを結合
        ↓
[AIエンジンに送信して画像生成]
        ↓
[生成されたスプライトシートを返却]
```

### 3.2 サポートする生成エンジン

| エンジン | 種別 | 用途 |
|---|---|---|
| **Gemini (Imagen 3)** | クラウド | 標準エンジン。`imagen-3.0-generate-002` 等を使用 |
| **Gemini Native Image (Nano Banana)** | クラウド | `flash-image` / `pro-image` 系モデル。画像を入力として渡せるマルチターン対話生成に対応 |
| **OpenAI (DALL-E)** | クラウド | `dall-e-3` / `dall-e-2` を使用 |
| **ComfyUI** | ローカル | ComfyUI ワークフローへのプロンプト送信（`ノードID:フィールドキー` 形式で指定） |
| **Ollama** | ローカル | 画像生成非対応のため使用不可 |

### 3.3 生成プロンプト構成

プロンプトは `prompt.yaml`（[prompt.yaml](../../ui/src/skills/expression-service/prompt.yaml)）で管理される。

```
最終プロンプト = common + ユーザープロンプトテンプレート + labels
```

| プロンプトキー | 内容 |
|---|---|
| `common` | キャラクターの一貫性を保つための共通指示 |
| `user-template-base` | ユーザーが編集可能なデフォルトテンプレート |
| `labels` | 各表情パネルの下に英語ラベルを表示させる指示 |
| `analyze-sprite-sheet` | スプライトシート解析時の指示（Gemini Vision 用） |

**デフォルトプロンプトテンプレート（ユーザー編集可能）：**

```
顔のアップで、チャットマスコットとして必要な様々な表情を作成してください。
作成する表情は以下の通りです：
[__EMOTIONS_LABLE__]
背景は純白(#ffffff)にしてください。
```

プレースホルダー `__EMOTIONS_LABLE__` は選択した感情の英語ラベルカンマ区切りリストに自動置換される。

### 3.4 Nano Banana シリーズ（Native Image モデル）の特徴

モデル名に `flash-image`、`pro-image`、または `banana` を含む場合は Native Image モデルとして扱われ、以下の挙動となる。

- ベースキャラクター画像をインラインデータとして一緒に送信し、画像の一貫性を保持する
- **マルチターン対話**に対応し、生成後に追加指示（修正依頼）を続けて入力できる
- レスポンスには会話履歴（`history`）が含まれ、次回のリクエストにコンテキストとして渡される

### 3.5 UI からの操作手順

1. 設定画面 → マスコット詳細設定 → 表情アセット タブを開く
2. **「AI表情生成」** ボタンをクリックして `AiExpressionGeneratorModal` を起動する
3. 生成する表情を選択（最大16個）
4. 必要に応じてプロンプトを調整する
5. **「AIスプライトを生成」** ボタンをクリックして生成を実行する
6. プレビューを確認後、**「AIスプライトをインポート」** をクリックして解析・割り当てフローへ進む

## 4. スプライトシートの表情割り当て

### 4.1 自動解析フロー（Gemini Vision）

スプライトシートのインポートは `importFromSpriteSheet` 関数（[MascotSettings.vue](../../ui/src/components/settings/MascotSettings.vue)）が処理する。

```
[スプライトシート画像の選択 or 生成結果の受け取り]
        ↓
[Gemini Vision API で解析]
  prompt: "Analyze this expression sprite sheet. Identify each facial expression box
           and return JSON with emotion label and coordinates [ymin, xmin, ymax, xmax] (0-1000)."
        ↓
[JSON レスポンスを受信]
  [{ label: "joy", box_2d: [ymin, xmin, ymax, xmax] }, ...]
        ↓
[各ボックスを Canvas で切り抜き]
  正規化座標（0-1000）→ 実ピクセル座標に変換
  canvas.toDataURL('image/png') で Base64 に変換
        ↓
[英語ラベルを日本語ラベルに変換]
  例: "joy" → "喜び"、"neutral" → "通常"
        ↓
[対応する感情スロットに自動割り当て]
  スロット名の完全一致で自動マッチング
        ↓
[感情割り当て画面（EmotionAssignmentModal）で確認・調整]
```

### 4.3 手動での表情割り当て

**表情エディタ** (`ExpressionEditorModal.vue`) から手動で各スロットに画像を割り当てることも可能。

- **画像から切り出し** — 既存画像ファイルを指定し、クロップモーダルで切り出して割り当て
- **切り抜き直す** — 登録済み画像をクロップしなおす（`originalPath` から元画像を復元）
- **表情解除** — スロットに登録されている画像を解除する（スロット自体は維持される）

### 4.4 表情スロットのデータ構造

表情は `MascotAsset` インターフェース（[mascot-asset.ts](../../ui/src/mascots/mascot-asset.ts)）で表現される。各スロットは衣装（Outfit）に紐付けられる。

```typescript
interface MascotAsset {
    id: string;         // スロットの一意ID
    name: string;       // 感情ラベル（日本語）例: "喜び"
    path: string;       // 画像パスまたは Base64 Data URL
    originalPath?: string; // クロップ元の元画像パス
    offsetX?: number;   // 横方向オフセット (px)
    offsetY?: number;   // 縦方向オフセット (px)
    scale?: number;     // 拡大率 (1.0 = 等倍)
}
```

## 5. 表情の位置合わせ

### 5.1 概念

表情画像は、マスコット本体（衣装またはポーズ画像）の上に**レイヤーとして重ね合わせ**て表示される。  
表情画像の位置と大きさは、各スロットが持つ `offsetX`、`offsetY`、`scale` パラメータで制御する。

### 5.2 位置調整パラメータ

| パラメータ | 型 | デフォルト | 範囲 | 説明 |
|---|---|---|---|---|
| `offsetX` | number | 0 | -250 〜 250 (px) | 横方向オフセット（正: 右、負: 左） |
| `offsetY` | number | 0 | -250 〜 250 (px) | 縦方向オフセット（正: 下、負: 上） |
| `scale` | number | 1.0 | 0.3 〜 2.0 | 拡大率（1.0 = 等倍） |

### 5.3 表情エディタでの調整方法

`ExpressionEditorModal.vue` で視覚的に調整できる。  
プレビューエリアはベースサイズ **420px × 560px** で表示される。

- **Y方向** — 縦スライダー（プレビューの右側に配置）で調整
- **X方向** — 横スライダー（下部コントロールパネル）で調整
- **スケール** — スライダー + 微調整ボタン（±0.01）で調整
- **リセット** — 「リセット」ボタンで `offsetX=0, offsetY=0, scale=1.0` に戻す

### 5.4 実際の CSS 変換

表情画像には以下のCSS `transform` が適用される。

```css
transform: translate(${offsetX}px, ${offsetY}px) scale(${scale})
```

表情画像の基本サイズは **140px × 140px**（`object-fit: contain`）で固定される。

### 5.5 エディタとデスクトップウィンドウのスケール補正

表情エディタのプレビューベースサイズ（420px）とデスクトップ上のマスコットウィンドウサイズ（512px）が異なるため、以下の補正係数が自動適用される。

```typescript
const scaleFactor = 512 / 420; // ≈ 1.219
const scaledOx = ox * scaleFactor;
const scaledOy = oy * scaleFactor;
```

これにより、エディタで調整した位置がデスクトップ上でも同じ見た目になるよう補正される。

### 5.6 設定画面のリスト内プレビューでのスケール補正

設定画面の小さなマスコットサムネイル（140px）でプレビューする際にも補正が適用される。

```typescript
const scaleFactor = 140 / 420; // = 1/3
```

## 6. 実行時の表情切り替えロジック

### 6.1 解決優先順位

`MascotViewer.vue` の `activeExpression` computed プロパティが現在表示すべき表情を以下の優先順位で解決する。

```
1. プレビュー中（expressionId 指定あり）
        ↓ プレビューIDで expressions から検索
2. neutral 状態 かつ defaultExpressionId が設定済み
        ↓ defaultExpressionId のスロットを使用
3. MascotImageSet から感情名でマッチング
        ↓ emotionFaceImages[currentEmotion]
        ↓ emotionFullbodyImages[currentEmotion]
4. 英語↔日本語変換マップで直接マッチング
        ↓ expressions から name の一致で検索
5. キーワードマッピングによるフォールバック
        ↓ happy/sad/angry/surprised/neutral のグループで検索
6. defaultExpressionId（再チェック）
        ↓
7. "通常"/"normal"/"neutral" キーワードで検索
        ↓
8. expressions[0]（最初のスロット）
```

### 6.2 キーワードグループマッピング

感情名が直接マッチしない場合、以下のグループにより「近い感情」の画像が選ばれる。

| グループ | 含まれる感情 |
|---|---|
| `happy` | 笑顔, 喜び, joy, amusement, excitement, love, admiration, approval, gratitude, optimism, pride, relief, 面白がり, 興奮, 愛情 等 |
| `sad` | 悲しみ, sadness, grief, remorse, disappointment, disapproval, 深い悲しみ, 後悔, 失望 等 |
| `angry` | 怒り, anger, annoyance, disgust, 苛立ち, 嫌悪 等 |
| `surprised` | 驚き, surprise, curiosity, realization, confusion, 好奇心, 気づき, 混乱 等 |
| `neutral` | 通常, neutral, caring, desire, embarrassment, fear, nervousness, 気遣い, 欲求, 当惑, 恐れ, 緊張 等 |
