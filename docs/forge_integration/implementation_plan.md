# Stable Diffusion WebUI Forge 連携 UI実装計画 (メニュー拡張＆チャット入力統合版)

本計画は、添付クリップボタンのメニューを拡張し、通常のチャット入力欄を「画像生成（t2i）/画像編集（i2i）モード」のプロンプト入力として使用できるようにし、かつロードしたモデル・LoRA一覧を設定画面および設定ダイアログでドロップダウン選択（datalist）可能にするための詳細設計を定義するものです。

---

## 1. 新たな要件とUI/UX設計

1. **アタッチメニューの4項目化**
   - 添付クリップボタンをクリックした際のメニューを以下の4つにします：
     - **ファイル添付**: 従来のローカルファイル選択を起動
     - **画像生成**: t2i (text-to-image) モードに遷移
     - **画像編集**: i2i (image-to-image) モードに遷移
     - **生成設定**: 画像生成パラメータの設定用ダイアログを表示

2. **チャット入力欄との統合 (t2i / i2i モード)**
   - 画像生成または画像編集モードが有効な場合、チャット入力欄（`inputText`）は通常のチャットメッセージ送信ではなく、**画像生成のプロンプト入力**として機能します。
   - 入力欄のプレースホルダーをモードに合わせて変更します：
     - t2i モード時: `"[画像生成モード] 生成したいプロンプトを入力..."`
     - i2i モード時: `"[画像編集モード] 編集の指示（プロンプト）を入力..."`
   - 入力欄の上部または横に、現在のモードを示すインジケーターと、**「モードを解除（チャットに戻る）」ボタン**を表示します。
   - モード中の送信ボタンクリック（または Enter）時に、選択中の画像生成エンジン（Forge / DALL-E）へ生成リクエストを送り、完了後に結果の画像をアタッチメント（`pendingAttachments`）に追加します。
   - **画像編集 (i2i)** 時は、アタッチメント欄にすでに存在する1枚目の画像を元画像（`init_images`）として Forge API に渡して生成を実行します（画像が添付されていない場合は警告を表示します）。

3. **生成設定ダイアログの追加**
   - `ForgeImageGeneratorDialog.vue` を改修し、即座の生成ではなく、**「生成パラメータ（エンジン、モデル、LoRA、縦横サイズ、ステップ数、CFG Scale）を一時保存・適用する設定ダイアログ」** に変更します。
   - ユーザーはここで設定を保存したのち、チャット欄から画像生成・編集を行います。

4. **モデル一覧・LoRA一覧のドロップダウン連携 (追加)**
   - **画像AI設定画面 (`ImageGenSettingsPanel.vue`)**:
     - 「モデル取得」「LoRA取得」ボタンを押して取得されたアセット一覧を `configStore` の `forgeModelsList` および `forgeLorasList` に格納し、`saveConfig` で永続化します。
     - デフォルトモデルおよびデフォルトLoRAの入力欄に `<datalist>` を適用し、取得済みのリストがある場合はドロップダウン候補から選択可能にします（手動入力も可能）。
   - **パラメータ設定ダイアログ (`ForgeImageGeneratorDialog.vue`)**:
     - 設定ダイアログでも、`forgeModelsList` および `forgeLorasList` を使用して、使用モデルおよびLoRA名入力欄に `<datalist>` を適用し、取得済みのリストからドロップダウン候補として選択可能にします。

## 2. 変更・追加ファイル一覧

### 2.1. 設定データ (`app/src/config/config-data.ts` & `store/config.ts`)
以下の画像生成パラメータの設定項目を追加・保持できるようにします：
- `forgeSteps`: number (デフォルト `25`)
- `forgeCfgScale`: number (デフォルト `7.0`)
- `forgeWidth`: number (デフォルト `1024`)
- `forgeHeight`: number (デフォルト `1024`)

### 2.2. コネクタ (`app/src/connector/forge-connector.ts`)
- `generateImage` メソッドに `initImage` (Base64形式の元画像) パラメータを追加し、元画像がある場合は `/sdapi/v1/img2img` エンドポイントを呼び出すように拡張します。

### 2.3. IPC定義 (`app/electron/main.ts`)
- img2img 生成の対応に伴い、メインプロセスの `forge:generate` ハンドラを引数に応じて i2i API にも振り分けられるように対応。

### 2.4. 設定UI & チャットUI
- **`ForgeImageGeneratorDialog.vue`**: 
  - パラメータ変更の適用ダイアログにリネーム・改修。
  - `<datalist>` によるモデル・LoRAの選択候補表示機能を追加。
- **`ImageGenSettingsPanel.vue`**:
  - モデル・LoRA取得成功時に `configStore` の一覧データを更新・保存する処理を追加。
  - `<datalist>` によるモデル・LoRAの選択候補表示機能を追加。
- **`ChatPanel.vue`**:
  - `imageGenMode`（`'t2i' | 'i2i' | null`）の状態管理。
  - モード中のプロンプト入力および生成処理の起動（生成ローディング中はインジケーター表示）。
  - クリップメニューを4項目に拡張。

## 3. 検証計画
- `vitest` によるコネクタ（i2i API 呼び出し）のユニットテスト追加。
- アプリを起動し、チャット入力欄での t2i / i2i 画像生成・編集フロー、およびパラメータ設定が正しく反映されるか動作テスト。
- 設定画面で「モデル取得」「LoRA取得」を実行し、設定画面およびチャットパラメータダイアログ双方でドロップダウン候補が表示され、正しく選択・保存できるか確認。
